# LLM SQL Local Benchmark 本地部署与局域网私有化运行指南

本指南将指导您如何在本地或公司局域网内快速部署和运行 **LLM SQL Local Benchmark (大模型 SQL 本地评测面板)**。

通过本地部署，所有测试的完整日志、思维链追踪、生成的 SQL 和统计报告都将自动以 `CSV` 和 `Markdown` 文件格式保存在您本地服务器磁盘上，极大地方便了离线数据分析。

---

## 💡 本地运行的核心优势
1. **自动归档保存**：每跑完一轮，测试详情（CSV）会自动按批次时间戳（如 `trace_20260707203015`）分类存盘在本地服务器的 `trace_log/` 目录下。
2. **完整汇总输出**：全部测试结束后，后台会自动生成 `summary.md` 测试汇总报告保存在同一目录下。
3. **支持打包下载**：在页面上点击“导出”时，后端会将该批次的所有 CSV 轨迹和汇总报告在服务端压缩打包，直接下发为 `.zip` 文件到浏览器。
4. **完全私有化**：无须依赖任何外部托管服务（如 GitHub Pages），适合在公司内网、无网开发环境部署。

---

## 🛠️ 环境准备
* **操作系统**：Windows / macOS / Linux
* **依赖环境**：只需要安装 **Python 3.x** 即可（项目的所有脚本仅使用 Python 的标准库，**免安装任何第三方依赖包**，如 `pip` 依赖）。

---

## 🚀 快速部署三步走

### 第一步：克隆/下载本仓库代码到本地
通过 Git 将代码拉取到您的本地设备上：
```bash
git clone https://github.com/wangxian001/SQL_LLM_benchmark.git
cd SQL_LLM_benchmark
```
*(如果不使用 Git，也可以在 GitHub 页面点击 "Code -> Download ZIP" 下载压缩包，并解压到本地文件夹)*。

### 第二步：初始化并下载 AdventureWorks 数据库表
双击运行 `download_tables.py` 脚本，或者在终端中运行：
```bash
python download_tables.py
```
* **该脚本做的事情**：自动从线上拉取评测专用的 AdventureWorks 数据库表（CSV 压缩格式），在本地解压缩并创建 `tables/` 目录。
* **成功标志**：运行结束后，项目目录下会新增一个 `tables/` 文件夹，内含 `Customer.csv`、`Product.csv`、`Sales.csv` 等 7 张数据表。

### 第三步：启动本地评测服务器
在终端中运行 `run_server.py` 启动本地轻量级 HTTP Web 服务器：
```bash
python run_server.py
```
* 服务器启动后，会自动帮您在默认浏览器中打开测试主页面：
  `http://localhost:8000/sql_benchmark.html`
* **控制台会输出**：
  ```text
  Serving HTTP on 0.0.0.0 port 8000 (http://0.0.0.0:8000/) ...
  Opening browser at: http://localhost:8000/sql_benchmark.html
  ```

## 🌐 局域网内其他设备如何访问？
若需要在公司的其他测试机或手机浏览器上访问该评测面板，可以非常方便地进行共享：

1. **获取您的服务器局域网 IP 地址**（例如 `192.168.2.150`）：
   * Windows 下在 cmd 运行：`ipconfig` 查看 IPv4 地址。
   * Linux / macOS 下运行：`ifconfig` 或 `ip a` 查看网卡 IP。
2. **在其他设备的浏览器中直接访问**：
   `http://192.168.2.150:8000/sql_benchmark.html`
   *(请将 `192.168.2.150` 替换为您的实际服务器局域网 IP)*
3. **安全组/防火墙提示**：如果其他设备无法打开页面，请确保您的部署服务器防火墙已放行 **8000** 端口。

## 📂 本地保存日志目录说明
所有通过本地部署实跑的评测，数据将实时写入至以下位置：
* **存盘总目录**：项目根目录下的 `trace_log/` 文件夹。
* **批次子目录**：每次点击“开始测试”，系统都会为该测试任务生成唯一的开始时间戳目录，格式为 `trace_YYYYMMDDHHMMSS`。
* **文件清单**：
  * `round_1.csv`、`round_2.csv` ...：各轮测试的明细日志（包含生成的 SQL、Tokens、首行数据比对、以及详细的每步交互 Trace 调试日志）。
  * `summary.md`：评测汇总报告（含各轮正确率统计、耗时、模型以及 transposed 热力图矩阵表格）。

## ❓ 常见问题排查与解决

### 1. 运行 `download_tables.py` 报错或下载超时？
* **原因**：部分网络环境下访问远程数据源不稳定导致。
* **解决**：可以直接使用浏览器访问 [https://sql-benchmark.nicklothian.com/data/tables/](https://sql-benchmark.nicklothian.com/data/tables/)，手动下载 `Customer.csv.gz` 到 `Sales_Territory.csv.gz` 共 7 个压缩包，在本地新建一个名为 `tables` 的文件夹，将其全部解压缩并命名为 `.csv` 放入即可。

### 2. 本地自建版与在线 GitHub Pages 版有什么区别？
* 克隆到本地后，请务必使用 `sql_benchmark.html` 进行本地实跑。该文件完美集成了本地 Python 写入磁盘和一键 ZIP 后端打包功能。
* `index.html` 是专为在线 GitHub Pages 静态托管所设计的版本。它不调用 Python 接口，而是直接在浏览器纯前端进行 JSZip 压缩下载，避免在静态页面上产生网络 404 报错。

### 3. 如何更改服务端口号？
如果您本地 8000 端口已被占用，可以用文本编辑器打开 `run_server.py`，修改第 14 行的端口参数：
```python
PORT = 8080 # 将 8000 改为您期望的空闲端口
```
修改保存后重新运行 `python run_server.py` 即可。
